⚙️ Stage 1: Preprocessing

Preparing the image for analysis

📏 Image Resizing

What it does: Reduces image dimensions for faster processing.

Why we need it: A 12MP image (4000×3000 pixels) takes much longer to process. We resize to max 800px for speed.

# Resize while keeping aspect ratio max_dim = 800 scale = max_dim / max(height, width) small = cv2.resize(image, None, fx=scale, fy=scale)
💡 Key Point
We save the scale factor to convert detected corners back to original coordinates later.

🌫️ Gaussian Blur

What it does: Smooths the image by averaging each pixel with its neighbors using weighted averages.

The Gaussian Function
G(x,y) = (1 / 2πσ²) × e-(x² + y²) / 2σ²

Why we need it:

  • Removes noise - Camera sensors produce random noise that can be mistaken for edges
  • Prepares for edge detection - Canny requires pre-smoothing
  • Reduces false positives - Without blur, every tiny texture looks like an "edge"

5×5 Gaussian Kernel

14741 41626164 72641267 41626164 14741

Center pixel (41) has highest weight

# Apply Gaussian blur with 5×5 kernel blurred = cv2.GaussianBlur(image, (5, 5), sigmaX=0)

🎨 Color Space Conversion

What it does: Converts image from one color representation to another.

BGR

Blue-Green-Red. How OpenCV stores color images. Each pixel = 3 values (0-255).

Grayscale

Single channel (0-255). Formula: 0.299×R + 0.587×G + 0.114×B

LAB

L=Lightness, A=Green-Red, B=Blue-Yellow. Perfect for shadow removal!

🌓 Stage 2: Shadow Removal

Gradient-Domain Processing in LAB Color Space

❓ Why LAB Color Space?

In BGR, brightness is mixed into ALL three channels. You can't tell if a pixel is dark because of shadow or because it's a dark color.

The Problem with BGR:

Dark red pixel: B=0, G=0, R=100 Is it dark because: ❓ It's in a shadow? OR ❓ It's genuinely dark red? We can't tell in BGR!

LAB Solution:

L Channel

ONLY Brightness

0 = Black, 100 = White

A Channel

ONLY Green↔Red

-128 to +127

B Channel

ONLY Blue↔Yellow

-128 to +127

✓ We can fix L (shadows) without touching A,B (colors)!

📐 Sobel Operators (Gradient)

Computes the rate of change of pixel intensity. High gradient = edge or shadow boundary.

Gx (Horizontal)

-10+1 -20+2 -10+1

Detects vertical edges │

Gy (Vertical)

-1-2-1 000 +1+2+1

Detects horizontal edges ─

Gradient Magnitude
G = √(Gx² + Gy²)
💡 How Convolution Works
For EVERY pixel in the image, we place the 3×3 kernel on top, multiply each kernel value by the corresponding pixel, sum them up, and that becomes the new pixel value. This is done with BOTH Gx and Gy kernels.

💡 Retinex Theory

The key insight: An image is the product of Reflectance and Illumination.

Core Equation
Image = Reflectance × Illumination

Reflectance

The actual surface properties (text, paper color) - what we WANT to keep

Illumination

The lighting conditions (shadows, uneven light) - what we want to REMOVE

# Retinex-based shadow removal # Step 1: Estimate illumination (large blur) illumination = cv2.GaussianBlur(l_channel, (31, 31), 0) # Step 2: Compute reflectance reflectance = l_channel / (illumination + 1e-6) # Step 3: Re-illuminate with UNIFORM light output = reflectance * np.mean(illumination)

📊 CLAHE

Contrast Limited Adaptive Histogram Equalization - Enhances contrast locally.

How It Works:

  1. Divide image into 8×8 tiles
  2. Equalize histogram in each tile separately
  3. Clip histogram to prevent over-enhancement
  4. Interpolate between tiles for smooth result

Parameters:

clahe = cv2.createCLAHE( clipLimit=2.0, # Max contrast tileGridSize=(8, 8) # 64 tiles ) enhanced = clahe.apply(l_channel)

🔍 Stage 3: Edge Detection

Finding document boundaries

⚡ Canny Edge Detection

The gold standard for edge detection. Produces thin, well-connected edge lines.

The 5 Steps:

  1. Gaussian Blur - Reduce noise
  2. Gradient Calculation - Sobel Gx, Gy
  3. Non-Maximum Suppression - Thin to 1px
  4. Double Thresholding - Strong/weak edges
  5. Hysteresis - Connect weak to strong

Adaptive Thresholds:

# Compute thresholds from image median = np.median(gray) lower = 0.7 * median upper = 1.3 * median edges = cv2.Canny(gray, lower, upper)

Canny

Binary output, thin edges, automatic noise suppression.

Clean Documents

Sobel

Gradient magnitude, shows edge "strength", good for textures.

Noisy Backgrounds

Laplacian

Second derivative, detects zero-crossings, very sensitive.

High Contrast

🔧 Stage 4: Morphological Operations

Cleaning up the edge detection results

Dilation

Expands white regions. Output = MAX of neighborhood.

dst(x,y) = max{src(x+i, y+j)}

✓ Closes gaps in edges
✓ Connects nearby objects

Erosion

Shrinks white regions. Output = MIN of neighborhood.

dst(x,y) = min{src(x+i, y+j)}

✓ Removes small noise
✓ Separates objects

Opening

Erosion → Dilation

Removes small bright spots (noise) while preserving larger shapes.

cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel)

Closing

Dilation → Erosion

Fills small holes inside objects, connects broken edges.

cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel)

📐 Stage 5: Contour & Corner Extraction

From edges to document boundaries

🔷 Douglas-Peucker Algorithm

Simplifies a contour to a polygon with fewer vertices while maintaining shape.

Epsilon Calculation
ε = 0.02 × perimeter(contour)

Algorithm:

  1. Draw line between first and last point
  2. Find point farthest from this line
  3. If distance > ε: split and recurse
  4. If distance ≤ ε: approximate as line
peri = cv2.arcLength(contour, True) epsilon = 0.02 * peri approx = cv2.approxPolyDP(contour, epsilon, True) if len(approx) == 4: # Found a quadrilateral! corners = approx.reshape(4, 2)

📍 Corner Ordering Algorithm

Ensures corners are in order: Top-Left → Top-Right → Bottom-Right → Bottom-Left

Sum (x+y): Difference (y-x): • Top-left: SMALLEST sum • Top-right: SMALLEST diff • Bottom-right: LARGEST sum • Bottom-left: LARGEST diff
def order_corners(corners): s = corners.sum(axis=1) # x + y d = np.diff(corners, axis=1) # y - x ordered[0] = corners[np.argmin(s)] # Top-left ordered[2] = corners[np.argmax(s)] # Bottom-right ordered[1] = corners[np.argmin(d)] # Top-right ordered[3] = corners[np.argmax(d)] # Bottom-left

🎯 Stage 6: Homography & Perspective Correction

The mathematical heart of document scanning

📊 The Homography Matrix

A 3×3 transformation matrix that maps points from one plane to another.

3×3 Matrix H

h₁₁h₁₂h₁₃ h₂₁h₂₂h₂₃ h₃₁h₃₂h₃₃

8 degrees of freedom (h₃₃=1)

Point Transformation
x' = (h₁₁·x + h₁₂·y + h₁₃) / (h₃₁·x + h₃₂·y + h₃₃)
y' = (h₂₁·x + h₂₂·y + h₂₃) / (h₃₁·x + h₃₂·y + h₃₃)

🧮 Direct Linear Transform (DLT)

Computes H from 4 point correspondences. Each point gives 2 equations → 8 equations for 8 unknowns.

# Source: detected document corners src = np.array([[100,80], [540,60], [560,400], [80,420]], dtype=np.float32) # Destination: perfect rectangle dst = np.array([[0,0], [width,0], [width,height], [0,height]], dtype=np.float32) # Compute homography using DLT internally H = cv2.getPerspectiveTransform(src, dst) # Apply perspective warp warped = cv2.warpPerspective(image, H, (width, height))

✨ Stage 7: Image Enhancement

Final polish for scanner-quality output

Unsharp Masking

Sharpens by enhancing edges.

sharp = img + α × (img - blur)
blurred = cv2.GaussianBlur(img, (0,0), sigmaX=1.5) sharp = cv2.addWeighted(img, 1.3, blurred, -0.3, 0)

Gamma Correction

Non-linear brightness adjustment.

out = 255 × (in/255)γ

γ < 1 → Brightens
γ > 1 → Darkens

🔇 Non-Local Means Denoising

Removes noise by averaging similar patches across the entire image (not just neighbors).

Why better than Gaussian: Preserves edges because edge pixels aren't similar to non-edge pixels.

denoised = cv2.fastNlMeansDenoisingColored( image, None, h=6, # Filter strength templateWindowSize=7, # Patch size searchWindowSize=21 # Search area )
