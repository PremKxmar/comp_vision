⚙️ Stage 1: Image Preprocessing

Preparing the image for analysis

🌫️ Gaussian Blur - Noise Reduction

What it does: Smooths the image by averaging each pixel with its neighbors using a weighted average. Closer pixels get more weight.

Why We Need It:

  • Removes noise - Camera sensors produce random noise that looks like edges
  • Prepares for edge detection - Canny edge detector requires pre-smoothing
  • Reduces false positives - Without blur, every tiny texture is detected as an "edge"

Before and after Gaussian blur with the 5×5 kernel weights shown

The Gaussian Function
G(x,y) = (1 / 2πσ²) × e-(x² + y²) / 2σ²

How It Works:

For each pixel, we place a kernel (e.g., 5×5) centered on it. Each neighbor is multiplied by its kernel weight, summed, and divided by total weight. The center pixel gets highest weight (41), edges get lowest (1).

# Apply Gaussian blur with 5×5 kernel blurred = cv2.GaussianBlur(image, (5, 5), sigmaX=0) # sigmaX=0 means OpenCV calculates sigma from kernel size

🎨 Bilateral Filter - Edge-Preserving Blur

What it does: A smarter blur that smooths textures while preserving edges.

The Problem with Gaussian Blur:

Regular Gaussian blur makes edges blurry too. If there's text near noise, Gaussian blurs both.

Bilateral Filter Solution:

It considers TWO things:

  1. Spatial distance - How far is the neighbor? (like Gaussian)
  2. Intensity difference - How similar is the neighbor's color? (NEW!)
💡 Key Insight
If a neighbor pixel is very different in color (like crossing an edge), its contribution becomes tiny. This preserves edges!
# Bilateral filter - preserves edges while smoothing # Parameters: diameter=11, sigmaColor=17, sigmaSpace=17 blurred = cv2.bilateralFilter(image, 11, 17, 17)

🌓 Stage 2: Shadow Removal

Gradient-Domain Processing in LAB Color Space

Why LAB Color Space?

In BGR/RGB, brightness is mixed into ALL three channels. You can't tell if a pixel is dark because of a shadow or because it's actually a dark color.

LAB separates brightness (L) from color (A, B) - perfect for shadow removal

The Three Channels:

Channel What It Contains Range
L ONLY Lightness/Brightness 0 (black) to 100 (white)
A ONLY Green↔Red color -128 to +127
B ONLY Blue↔Yellow color -128 to +127
💡 Magic of LAB
We can fix the L channel (remove shadows) and leave A, B untouched. Result: shadows removed, colors preserved!

📐 Sobel Operators - Gradient Computation

What it does: Computes the rate of change of pixel intensity. High gradient = edge or shadow boundary.

How Sobel convolution works: kernel × pixels → sum → gradient value

The Two Sobel Kernels:

Gx (Horizontal)

-10+1 -20+2 -10+1

Detects vertical edges │

Gy (Vertical)

-1-2-1 000 +1+2+1

Detects horizontal edges ─

Gradient Magnitude Formula
G = √(Gx² + Gy²)

How Convolution Works (For EVERY Pixel):

For pixel at (5,5): 1. Extract 3×3 neighborhood around it 2. Multiply each pixel by corresponding Gx kernel value 3. Sum all products → Gx value 4. Repeat with Gy kernel → Gy value 5. Magnitude = √(Gx² + Gy²) This is done for EVERY pixel with BOTH kernels! For 1000×800 image = 1.6 million convolutions

💡 Retinex Theory - Illumination Normalization

The key insight: An image is the product of Reflectance (what we want) and Illumination (shadows - what we want to remove).

Image = Reflectance × Illumination → Divide out illumination to remove shadows

Core Equation
Image = Reflectance × Illumination
Reflectance = Image ÷ Illumination

Our Algorithm:

  1. Estimate illumination using large Gaussian blur (shadows are small-scale)
  2. Divide original by illumination → get reflectance
  3. Re-illuminate with uniform (mean) lighting
# Retinex-based shadow removal illumination = cv2.GaussianBlur(l_channel, (31, 31), 0) reflectance = l_channel / (illumination + 1e-6) output = reflectance * np.mean(illumination) # Uniform light

📊 CLAHE - Adaptive Contrast Enhancement

Contrast Limited Adaptive Histogram Equalization - Enhances contrast locally without over-amplifying noise.

How It Works:

  1. Divide image into 8×8 grid (64 tiles)
  2. Equalize histogram in each tile separately
  3. Clip histogram at limit to prevent over-enhancement
  4. Interpolate between tiles for smooth result
clahe = cv2.createCLAHE( clipLimit=2.0, # Max contrast amplification tileGridSize=(8, 8) # 64 tiles ) enhanced = clahe.apply(l_channel)

🔍 Stage 3: Edge Detection

Finding document boundaries

⚡ Canny Edge Detection - The Gold Standard

Produces thin, well-connected edge lines through a 5-step process.

The 5 steps of Canny edge detection

The 5 Steps Explained:

Step Operation Purpose
1 Gaussian Blur Reduce noise that could cause false edges
2 Gradient Calculation Sobel Gx, Gy → magnitude & direction
3 Non-Maximum Suppression Thin edges to 1 pixel width
4 Double Thresholding Classify as strong/weak/non-edge
5 Hysteresis Keep weak edges connected to strong ones
# Adaptive Canny thresholds median = np.median(gray) lower = int(max(0, 0.7 * median)) upper = int(min(255, 1.3 * median)) edges = cv2.Canny(gray, lower, upper)

🎛️ Adaptive Thresholding

What it does: Converts grayscale to binary, calculating threshold locally for each region.

Why Not Global Thresholding?

Documents have uneven lighting. A threshold that works for one corner may fail in another.

binary = cv2.adaptiveThreshold( gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C, # Gaussian weighted mean cv2.THRESH_BINARY, blockSize=11, # Neighborhood size (must be odd) C=2 # Constant to subtract )

🔧 Stage 4: Morphological Operations

Cleaning up the edge detection results

The Four Core Operations

Dilation, Erosion, Opening, and Closing operations on binary images

Dilation (Expand)

Output = MAX of neighborhood

  • Closes gaps in edges
  • Connects nearby objects
  • White regions grow outward

Erosion (Shrink)

Output = MIN of neighborhood

  • Removes small noise dots
  • Separates connected objects
  • White regions shrink inward

Opening (Erosion → Dilation)

Removes small bright spots while preserving larger shapes.

cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel)

Closing (Dilation → Erosion)

Fills small holes and connects broken edges.

cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel)

📐 Stage 5: Contour & Corner Extraction

From edges to document boundaries

🔷 Douglas-Peucker Polygon Approximation

What it does: Simplifies a contour with hundreds of points to a polygon with just 4 corners.

From 50+ contour points to just 4 corner vertices

Epsilon Calculation
ε = 0.02 × perimeter(contour)

Algorithm Steps:

  1. Draw line between first and last contour point
  2. Find the point farthest from this line
  3. If distance > ε: split at that point and recurse
  4. If distance ≤ ε: approximate as straight line
peri = cv2.arcLength(contour, True) epsilon = 0.02 * peri # 2% of perimeter approx = cv2.approxPolyDP(contour, epsilon, True) if len(approx) == 4: corners = approx.reshape(4, 2) # Found a quadrilateral!

📍 Corner Ordering Algorithm

Ensures corners are in consistent order: Top-Left → Top-Right → Bottom-Right → Bottom-Left

Using sum and difference to identify each corner

Corner Position Rule
Top-Left (0) Small x, small y Minimum sum (x+y)
Top-Right (1) Large x, small y Minimum difference (y-x)
Bottom-Right (2) Large x, large y Maximum sum (x+y)
Bottom-Left (3) Small x, large y Maximum difference (y-x)

🎯 Stage 6: Homography & Perspective Correction

The mathematical heart of document scanning

What is Homography?

A homography is a transformation that maps points from one plane to another. It "reverses" the perspective distortion caused by taking a photo at an angle.

From distorted trapezoid to perfect rectangle using the 3×3 homography matrix

📊 The Homography Matrix

A 3×3 matrix with 8 degrees of freedom that defines the complete transformation.

3×3 Homography Matrix H

h₁₁h₁₂h₁₃ h₂₁h₂₂h₂₃ h₃₁h₃₂h₃₃
Point Transformation Equations
x' = (h₁₁·x + h₁₂·y + h₁₃) / (h₃₁·x + h₃₂·y + h₃₃)
y' = (h₂₁·x + h₂₂·y + h₂₃) / (h₃₁·x + h₃₂·y + h₃₃)

🧮 Direct Linear Transform (DLT)

Computes H from 4 point correspondences. Each point gives 2 equations → 8 equations for 8 unknowns.

# Source: detected document corners (trapezoid) src = np.array([[100,80], [540,60], [560,400], [80,420]], dtype=np.float32) # Destination: perfect rectangle dst = np.array([[0,0], [width,0], [width,height], [0,height]], dtype=np.float32) # Compute homography using DLT internally H = cv2.getPerspectiveTransform(src, dst) # Apply perspective warp warped = cv2.warpPerspective(image, H, (width, height))
💡 Bilinear Interpolation
When warping, most output pixels don't land exactly on input pixels. We interpolate from the 4 nearest neighbors weighted by distance.
